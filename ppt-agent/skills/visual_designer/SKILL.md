# Visual Designer Skill

在动手之前，先像一个资深设计师一样思考。好的设计不是"找模板"，而是对"这个界面要解决什么问题、谁是用户、什么让它令人难忘"有清晰答案。

---

## 设计哲学（Design Philosophy）

设计前必须回答这三个问题，回答的质量决定设计的上限：

**Purpose**：这个PPT解决什么问题？给谁看？
- 给投资人看 → 强调数据说服力，视觉要大气
- 给客户提案 → 强调差异化，视觉要有记忆点
- 给内部培训 → 强调清晰易读，视觉要克制

**Tone**：选择一个极端的审美方向
- 科技前沿 → 深空蓝 + 科技青，拒绝渐变滥用
- 商务正式 → 深海蓝 + 橙强调色，拒绝花哨字体
- 极简美学 → 黑白灰为主，拒绝多余装饰
- 创意表达 → 大胆撞色 + 独特排版，拒绝模板感

**Differentiation**：什么让这份PPT"令人过目不忘"？
- 一个独特的视觉符号贯穿全篇？
- 一个让人眼前一亮的配色方案？
- 一种突破常规的布局方式？

## 内容质量评估（前置检查）

在生成设计之前，必须检查内容是否满足最低要求：

| 页面类型 | 最低要点数 | 每要点最少字数 | 最多要点数 |
|---------|-----------|--------------|-----------|
| `title_slide` | 1 | 标题 4 字 | 1 |
| `content_slide` | 3 | 20 字 | 7 |
| `two_column` | 每栏 2 个要点 | 每要点 25 字 | 每栏 4 个 |
| `section_divider` | 章节标题 6 字 | — | 1 |
| `summary_slide` | 3 | 15 字 | 7 |

### 内容增强规则

如果内容不满足要求，**先增强再设计**：

1. 如果要点 < 最低要求，用大模型扩写每个要点到 20-30 字，添加具体例子或数据
2. 如果没有图片描述，补充相关配图关键词（格式：`"概念名" → ["图片搜索关键词1", "关键词2"]`）
3. 如果是数据页，确保每个数据点都有具体数值和来源

## NEVER 清单（绝对禁止）

**颜色**
- 禁止使用紫色渐变 + 白底（最典型的"AI感"配色）
- 禁止使用 Inter、Roboto、Arial 作为主要英文字体
- 禁止使用超过 3 种主色调
- 禁止使用彩虹色渐变

**排版**
- 禁止同一层级的内容使用不同的对齐方式（左对齐就全是左对齐）
- 禁止字号混乱（标题和正文之间不能有中间值）
- 禁止行间距 < 1.2x 或 > 2x

**布局**
- 禁止使用"三栏等分 + 图标"这种模板感极强的布局
- 禁止内容页使用全屏背景图 + 半透明文字
- 禁止每页都塞满装饰元素（呼吸感同样重要）
- 禁止页面边缘没有留白

**内容**
- 禁止空洞的要点（少于 20 字的要点视为空洞）
- 禁止要点之间逻辑混乱或重复
- 禁止标题和内容牛头不对马嘴

## 色彩系统（4 种精选主题）

每种主题给出完整的语义化色彩变量，确保视觉一致性：

### 商务正式 (professional)

```json
"color_system": {
  "primary": "#1A365D",
  "primary_light": "#2B6CB0",
  "accent": "#63B3ED",
  "background": "#FFFFFF",
  "background_alt": "#F7FAFC",
  "text_main": "#1A202C",
  "text_secondary": "#4A5568",
```

适用：企业汇报、财报、投资人汇报。强调色用 `#FF6B35` 可增加活力。

## 字体与字号层级

### 推荐字体

| 风格 | 中文标题 | 中文字体 | 英文字体 | 适用场景 |
|------|---------|---------|---------|---------|
| 现代科技 | 思源黑体 Heavy | 思源黑体 Regular | Inter | AI、科技、产品发布 |
| 商务正式 | 方正兰亭黑 Bold | 微软雅黑 | Segoe UI | 企业汇报、财报 |
| 创意艺术 | 站酷快乐体 | 思源柔黑 | Poppins | 创意提案 |
| 极简主义 | 思源宋体 Bold | 思源黑体 Light | Playfair Display + Lato | 学术、艺术 |

### 字号层级（递减原则）

| 层级 | 字号范围 | 行高 | 用途 |
|------|---------|------|------|
| 主标题 | 40-48pt | 1.2x | 封面标题 |
| 章节标题 | 32-36pt | 1.3x | 内容页标题 |
| 副标题 | 24-28pt | 1.3x | 小标题、分隔页 |
| 正文要点 | 18-20pt | 1.5x | 主要内容 |
| 辅助说明 | 14-16pt | 1.4x | 注释、来源 |
| 页码/日期 | 12pt | 1.0x | 底部信息 |

## 布局决策树

根据要点数量和内容类型快速路由：

### 按要点数量

| 要点数量 | 推荐布局 | 说明 |
|---------|---------|------|
| ≤ 3 条 | `centered_bullets` | 居中，每条加大字号 + 图标 |
| 4-6 条 | `two_column_bullets` | 分两列，节省空间 |
| ≥ 7 条 | `grouped_bullets` | 分组 + 小标题 + 子要点 |
| 含数据对比 | `left_text_right_chart` | 左侧文字，右侧图表 |

### 按内容类型

| 内容类型 | 推荐布局 | 视觉特征 |
|---------|---------|---------|
| 概念/定义 | 左图右文 | 左侧图标，右侧文字说明 |
| 多维度对比 | 双栏对比 | 左右分栏，中间分隔线用 accent 色 |
| 时间序列 | 时间轴 | 横向排列，用连接线串联 |
| 步骤流程 | 分步图 | 箭头连接，每步配图标 |
| 金句/引言 | 大字引言 | 72pt+ 大字居中，背景装饰 |
| 数据展示 | 图表优先 | 数据可视化为主，文字辅助 |

## 视觉元素使用原则

**宁少勿多，宁精勿滥。** 每页至少有一个视觉元素，但不要让装饰喧宾夺主。

### 图标匹配规则

根据内容关键词自动匹配，不要随机使用：

| 关键词类型 | 推荐图标 |
|-----------|---------|
| 概念/定义 | 📖 💡 🎯 |
| 数据/统计 | 📊 📈 📉 |
| 流程/步骤 | ⚙️ 🔄 ⏩ |
| 优势/特点 | ✅ ✨ ⭐ |
| 问题/挑战 | ⚠️ 🚧 |
| 技术/实现 | ⚡ 🔧 💻 |
| 结果/成果 | 🏆 💪 |
| 未来/趋势 | 🚀 🌟 |

### 装饰元素（克制使用）

每页最多 1-2 个装饰元素：
- 顶部装饰线：标题下方，accent 色，3pt 高
- 底部装饰条：页面底部，渐变色块，8px 高
- 侧边几何：右侧浅色块，仅用于标题页

## 页面节奏规划

### 整体结构模板

```
第 1 页：标题页 (title_slide) — 封面信息
第 2 页：目录/概览页（可选）
第 3 ~ N-2 页：内容页循环
  - 每 2-3 页内容后插入 1 页分隔页（视觉呼吸点）
  - 偶数页使用 background_alt 交替背景
第 N-1 页：总结页 (summary_slide)
第 N 页：结束页 (thank_you)
```

### 视觉节奏规则

- 每 3 页插入一个分隔页作为"视觉呼吸点"
- 偶数页使用 `background_alt` 增加变化
- 每 5 页插入一张高质量图片作为视觉高潮

## 常见场景快速决策

### 配色选择
- 面向投资人/高管 → `professional`，强调色用 `#FF6B35`
- 面向技术团队 → `tech`，深空蓝 + 科技青
- 面向学术会议 → `professional`，深海蓝
- 面向年轻用户/产品发布 → `creative`，紫色系
- 追求极简美学 → `minimal`，黑白灰

### 布局选择
- 概念解释 + 数据 → 双栏对比 + 数据图表
- 单一概念深挖 → 内容页 + 大量图标
- 时间/阶段展示 → 时间线布局
- 核心观点强调 → 引言布局 + 背景装饰
- 流程步骤展示 → 分步图 + 箭头连接

## 内容增强工作流

当上游 Planner 提供的内容不够充实時，执行以下增强流程：

1. **扩写要点**：如果要点 < 20 字，扩写到 20-30 字，添加具体例子、数据、解释
2. **添加视觉资产**：为每个概念生成 1-2 个配图搜索关键词
3. **生成图表数据**：如果是数据对比页，生成 3-5 个数据点
4. **补充背景信息**：调用搜索工具获取相关定义、数据，确保每页有足够的信息量
