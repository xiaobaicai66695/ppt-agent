# Visual Designer

## 色彩纯度强制要求（所有配色必须遵守）

**禁止使用刺眼的"原色"，必须使用低饱和度的灰感色彩。**

### 硬性禁止

| 类别 | 禁止颜色示例 | 原因 |
|------|-------------|------|
| 纯色原相 | `#FF0000` `#00FF00` `#0000FF` `#FFFF00` | 饱和度100%，刺眼 |
| 霓虹色 | `#FF00FF` `#00FFFF` `#FF6600` | 高饱和度，廉价感 |
| 高饱和强调色 | 任何RGB中有两通道接近0或255 | 生硬、缺乏质感 |
| 紫色渐变 | 紫色到蓝色/粉色的渐变 | 典型"AI感"配色 |

### 灰度混入原则

所有颜色的 RGB 三通道间差值应小于 200，且至少有一个通道值 < 200。

**验证方法**：将颜色视为立方体坐标，三个通道值越接近，颜色越"灰"，越柔和。

### 低饱和度配色标准

| 参数 | 允许范围 | 推荐范围 |
|------|---------|---------|
| 饱和度 (S) | 5% ~ 45% | 15% ~ 35% |
| 亮度 (B/L) 深色 | 35% ~ 55% | 40% ~ 50% |
| 亮度 (B/L) 浅色/背景 | 80% ~ 95% | 85% ~ 92% |

### 莫兰迪色系参考

推荐使用带灰感的"莫兰迪色系"：

| 用途 | 推荐颜色 | 替代高饱和色 |
|------|---------|-------------|
| 深色主色 | `#1B2A3A` `#2D3E50` `#3A4F6B` | 替代 `#0000FF` `#0066CC` |
| 灰蓝强调 | `#5899A8` `#6B8E9B` `#7FA9C7` | 替代 `#00B4D8` `#3498DB` |
| 灰紫主色 | `#6B4E6E` `#7D6080` | 替代 `#7C3AED` |
| 灰粉点缀 | `#C4A5B2` `#D4B8B8` | 替代 `#F59E0B` `#FF6B35` |
| 灰青辅助 | `#5A7A85` `#6B8E9B` | 替代 `#6366F1` |

## 配色原则

**颜色需有情绪意图，而非机械套用规则。**

- **判断分寸**：检查当前配色是否与内容调性匹配。科技主题可用冷蓝传递专业感，也可用暖橙传递活力——选择取决于你想让观众感受到什么。
- **情绪匹配**：严肃汇报用低饱和度配色，创意展示可用高饱和度点缀。颜色是情感语言，不是安全检查表。
- **克制用色**：主色调不超过 2 种，辅助色用于层次区分。当你想强调某处时，问自己"这块颜色在说什么"。
- **留白的呼吸感**：留白不是浪费空间，是给内容留出呼吸余地。空白处让视觉有落点。

**关于"AI感"的辨识**：
AI感配色往往表现为：紫色渐变 + 白底的"科技风"模板感、用色均匀到缺乏层次、饱和度机械统一。这些不是错误，但会让PPT失去个性。当你意识到自己在套用而非选择时，就是调整的时机。

## 字体原则

中英文字体组合使用，同一页面层级统一字体家族，避免混用。

### 中文字体

| 用途 | 推荐字体 |
|------|---------|
| 标题 | 思源黑体 / 微软雅黑 / 苹方 |
| 正文 | 思源黑体 / 微软雅黑 / 苹方 |

### 英文字体配对

选择有个性的标题字体搭配干净的正文/正文字体：

| 标题字体 | 正文字体 | 风格 |
|---------|---------|------|
| Georgia | Calibri | 经典优雅 |
| Arial Black | Arial | 现代简洁 |
| Calibri | Calibri Light | 商务干净 |
| Cambria | Calibri | 传统专业 |
| Trebuchet MS | Calibri | 科技感 |
| Palatino | Garamond | 文学气质 |

### 字号规范

| 元素 | 字号 |
|------|------|
| 幻灯片标题 | 36-44pt 加粗 |
| 章节标题 | 20-24pt 加粗 |
| 正文文字 | 14-16pt |
| 注释/来源 | 10-12pt 淡化色 |

## 动态配色方案

配色不是固定的4个主题，而是根据内容关键词动态生成。可调节"风格倾向"参数：

| 风格参数 | 说明 | 适用场景 |
|---------|------|---------|
| `temperature: cool` | 冷色调为主，专业沉稳 | 技术文档、学术汇报 |
| `temperature: warm` | 暖色调点缀，有温度感 | 团队分享、用户案例 |
| `temperature: neutral` | 低饱和度，稳定克制 | 商务正式报告 |

**生成示例（已灰化处理，底色为柔和肉色）**：
```
// 主题: AI技术演讲, 风格: cool（低饱和灰蓝）
primary: #1B2A3A, accent: #5899A8, bg: #FAF0E6, alt: #F0E4D4

// 主题: 创业故事分享, 风格: warm（低饱和灰粉）
primary: #5A4A5A, accent: #B8A0A8, bg: #FBF0E8, alt: #EDE0D4

// 主题: 产品发布会, 风格: neutral（低饱和灰调）
primary: #2D3E50, accent: #6B8E9B, bg: #FAF0E6, alt: #F0E8DC
```

### 颜色灰化指引（LLM 动态生成时使用）

给定一个"主题意向色"，将其转换为柔和版本：
1. 将 RGB 转为 HSL
2. 将饱和度 S 降低 30%~50%
3. 将亮度 L 调整至 35%~65%（深色）或 80%~92%（背景）
4. 转回 RGB 输出

**示例转换**：
- 意向"科技蓝 #0066CC" → 降低饱和度后 → `#4477AA`
- 意向"热情橙 #FF6600" → 转为灰感 → `#C47A5A`
- 意向"商务蓝 #1A56DB" → 灰化 → `#3A6B8C`

## 设计 Idea（拒绝无聊的幻灯片）

**不要做无聊的幻灯片。** 纯文字 + 白底的标题列表无法打动任何人。为每一页考虑以下创意。

### 开始前必做

- **选定大胆的、与内容契合的配色方案**：配色应该为这个主题量身定制。如果把颜色换到另一个完全不同的演示文稿上仍然"能用"，说明你的选择还不够具体。
- **主次分明**：一种颜色应占据主导（60-70% 视觉权重），配合 1-2 种辅助色和一种锐利的强调色。切忌所有颜色平分秋色。
- **统一使用浅色基调**：所有幻灯片统一使用浅色背景（柔和肉色、米白、浅灰等），营造明亮清爽的视觉感受。避免使用深色背景。
- **视觉主题贯穿始终**：选择一个独特元素并在每页重复使用——圆角图片边框、带颜色的圆形图标、一侧细边框等。

### 排版间距规范

| 元素 | 字号 |
|------|------|
| 幻灯片标题 | 36-44pt 加粗 |
| 章节标题 | 20-24pt 加粗 |
| 正文文字 | 14-16pt |
| 注释/来源 | 10-12pt 淡化色 |

- 页边距至少 0.5 英寸
- 内容块间距 0.3-0.5 英寸
- 不要填满每一寸空间，保持呼吸感

### 每页幻灯片都要有视觉元素

**纯文字幻灯片容易被人遗忘。**

布局选项：
- 双栏（左侧文字，右侧插图）
- 图标 + 文字行（图标放在带颜色的圆形中，粗体标题，下方说明）
- 2x2 或 2x3 网格
- 半出血图片（占满左/右半边），叠加文字内容

数据展示：
- 大数字强调（大号数字 60-72pt，下方小标签）
- 对比列（前后对比、优缺点对比）
- 时间线或流程（带编号步骤和箭头）

视觉修饰：
- 章节标题旁用小彩色圆形图标
- 关键数据用斜体强调

### 避免常见错误（必读）

| 错误 | 正确做法 |
|------|---------|
| 重复使用相同布局 | 变换使用双栏、卡片、大数字强调等多种布局 |
| 正文居中对齐 | 正文左对齐；仅标题居中 |
| 字号对比不足 | 标题需 36pt+ 才能从 14-16pt 正文中突出 |
| 默认用蓝色 | 选与主题相符的颜色 |
| 间距随意 | 选择 0.3" 或 0.5" 间距并保持一致 |
| 只做一页精致的，其余敷衍 | 全程保持统一水准，或全程保持简洁 |
| 纯文字幻灯片 | 添加图片、图标、图表或视觉元素 |
| 文本框无内边距 | 当文字与形状对齐时，设置 `margin: 0` |
| 低对比度元素 | 图标和文字都需要与背景形成强对比 |
| **标题下加装饰线** | **这是 AI 生成幻灯片的典型特征，禁止使用。用留白或背景色代替。** |

## 质量检查（QA）流程（必做）

**假设存在问题。发现它们是你的职责。**

初次渲染几乎不可能完全正确。把 QA 当作 bug 排查，而非确认步骤。如果第一次检查没发现任何问题，说明你看得不够仔细。

### 内容 QA

```bash
python -m markitdown output.pptx
```

检查内容是否缺失、有无错别字、顺序是否正确。

### 视觉 QA

**使用子代理进行视觉检查。** 即使只有 2-3 页也用子代理。你一直盯着代码，会看到你以为的样子，而非实际样子。子代理有新鲜视角。

将幻灯片转换为图片后（见下方转换方法），让子代理检查：

```
请对这些幻灯片进行视觉检查。假设存在问题——请找出它们。

检查要点：
- 重叠元素（文字穿透形状、线条穿过文字、堆叠元素）
- 文字溢出或被边缘/形状边界截断
- 装饰线位置为单行文字设计，但标题换行成了两行
- 来源引用或页脚与上方内容碰撞
- 元素间距太小（< 0.3"）或卡片/区块几乎相接
- 间距不均（一处大面积空白，另一处拥挤）
- 页边距不足（距边缘 < 0.5"）
- 列或类似元素对齐不一致
- 文字对比度低（如浅灰色文字配奶油色背景）
- 图标对比度低（如深色背景上的深色图标无对比圆形衬托）
- 文本框太窄导致换行过多
- 残留的占位符内容

列出每页的问题或可疑之处，即使是小问题也要报告。

分析这些图片：
1. /path/to/slide-01.jpg (预期: [简要描述])
2. /path/to/slide-02.jpg (预期: [简要描述])

报告所有发现的问题，包括小问题。
```

### 验证循环

1. 生成幻灯片 → 转换为图片 → 检查
2. **列出发现的问题**（如无问题，再仔细看一遍）
3. 修复问题
4. **重新验证受影响的幻灯片**——一个修复常会引发另一个问题
5. 重复直到完整检查后无新问题

**在完成至少一次修复-验证循环之前，不要宣告完成。**

### 转换为图片

将演示文稿转换为图片进行视觉检查：

```bash
python scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
```

生成 `slide-01.jpg`、`slide-02.jpg` 等文件。

修复后重新渲染特定幻灯片：

```bash
pdftoppm -jpeg -r 150 -f N -l N output.pdf slide-fixed
```

## 布局骨架选择

布局由**内容性质**驱动，而非要点数量决定。

| 内容性质 | 推荐骨架 | 说明 |
|---------|---------|------|
| **核心概念定义** | 居中意象 + 关键词强调 | 概念不宜套用"左图右文"，居中放置让焦点集中 |
| **多维度对比** | 双栏对称或卡片阵列 | 对比需要并置才有效果 |
| **发展历程/时间线** | 水平或垂直时间轴 | 时间感需要流动的视觉引导 |
| **操作步骤/流程** | 分步图或流程图 | 步骤需要清晰的前后关系 |
| **金句/核心观点** | 大字居中，充足留白 | 引言需要视觉冲击力，留白增强仪式感 |
| **数据展示** | 图表优先，图注清晰 | 数据页去装饰化，让图表说话 |
| **信息密集** | 分组 + 小标题，多级列表 | 信息多时用结构化组织，而非堆砌 |
| **并列要点** | 错落排列，打破机械对齐 | 多条内容不必整整齐齐，可左右错落、上下错位，增添活力 |

**布局的节奏感**：
- 排版不宜太过死板简约。多条并列时可以不并排排列，可以左右错开，让视觉有呼吸。
- 同一层级的元素不必强制对齐，可以通过缩进、色块大小、字号差异来体现层级关系。
- 卡片式布局中，各卡片高度可以不同，根据内容量自然撑开。
- 适当的不对称反而能引导视线流动。

**数量只是填充约束**：
- 内容少时不要撑满版面，用留白和字号体现重点
- 内容多时优先拆分，而非压缩到一屏
- 同一主题多个要点需详细解释 → 每点一页，大标题不变副标题变

### 空间布局软约束

以下规则帮助避免元素重叠、越界或失调。如与创意表达冲突，以设计效果为准，但需确保结果可正常渲染。

| 现象 | 软约束建议 |
|------|-----------|
| 元素超出幻灯片边界 | 所有元素应位于安全版心内（左右边距≥0.5英寸，上下边距≥0.4英寸） |
| 元素随意重叠 | 独立视觉元素间保持至少 0.15 英寸间距；仅半透明装饰与文字的叠加、明确图层关系描述例外 |
| 错落排列偏移失控 | 错落偏移量需给出具体数值，且不超出安全版心 |
| 内容塞太满挤在一起 | 内容超出版心容纳能力时，优先拆分为多页 |

定位描述参考示例：
- ✅ `“左下角（距左0.6英寸, 距下0.5英寸）放置圆形装饰”`
- ✅ `“标题下方0.2英寸处放置一条0.8英寸长的细线”`
- ❌ `“在角落放个圆形”`（缺乏具体位置和尺寸）

## 文字溢出防护（几何体内嵌文字必须遵守）

1. **先估算**：中文文字宽度 ≈ `字数 × 字号 × 0.5`，英文 ≈ `字符数 × 字号 × 0.3`
2. **几何体宽度 ≥ 文字宽度 + 两倍边距**，宁可大不可小
3. **优先换行**，换行后仍超出才缩小字号（最小 14pt）
4. **绝对不允许文字超出几何体边界**
5. **内容溢出版心的处理**：当单页内容无法满足间距要求且不减小字号（最小14pt）时，**优先拆分为多页**，而非强行压缩导致重叠或溢出。

## 设计输出格式

```json
{
  "theme": "tech",
  "style_temperature": "cool",
  "slides": [
    {
      "index": 1,
      "type": "title_slide",
      "content": { "title": "...", "subtitle": "..." },
      "style": { "primary_color": "#1B2A3A", "accent_color": "#5899A8" },
      "visual_elements": ["..."],
      "image_url": ""
    },
    {
      "index": 2,
      "type": "content_slide",
      "content": { "title": "...", "bullets": [{ "text": "..." }] },
      "style": { "primary_color": "#1B2A3A", "accent_color": "#5899A8", "line_height": "1.5x" },
      "visual_elements": ["..."],
      "image_url": "https://..."
    }
  ]
}
```

**颜色使用注意**：所有 `primary_color` 和 `accent_color` 必须使用低饱和度灰感色彩，参考上方"色彩纯度强制要求"章节。禁止在设计输出中使用高饱和度原色。

### visual_elements 字段说明

允许两种形式：

**1. 模板元素（精确控制）**
```json
"visual_elements": [
  "顶部渐变条: 高度0.3英寸, 颜色从primary到accent渐变",
  "右下角圆形装饰: 半径0.4英寸, accent色, 半透明0.3, 位置距右0.5英寸距下0.5英寸"
]
```

**2. 自然语言创意描述（发挥排版巧思）**
```json
"visual_elements": [
  "用几何色块构建抽象的数据流动感，左侧深右侧浅暗示趋势",
  "标题下方添加细线装饰，与内容区块形成视觉呼应",
  "底部角落用渐变圆形作为品牌视觉锚点，与顶部标题区形成对角线平衡"
]
```
