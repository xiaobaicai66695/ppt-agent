# 搜索规范指南

本文档是 visual_designer 的补充参考，按需查阅。

## 搜索原则

**网络搜索有 QPS/RPM 限流，优先使用模型已有知识，非必要不搜索。**

每个 PPT 任务搜索总调用次数建议不超过 10 次。

## 何时搜索

仅在以下情况使用搜索：
- 用户明确要求查找最新信息或数据
- 需要大模型不知道的具体数字、日期、统计数据（如某公司财报、特定年份数据）
- 需要核实大模型可能不确定的事实（如某产品发布时间、技术参数）
- 缺少必要的关键信息（如专业术语解释、事件时间线等）

## 何时不搜索（禁止浪费配额）

以下内容模型已掌握，**无需搜索**：
- 常见概念：CNN、Transformer、RNN、LSTM、注意力机制等基础技术
- 通用历史：深度学习发展脉络、ImageNet 竞赛、大模型进化时间线
- 常见算法原理：反向传播、梯度下降、Tokenization 等

## 搜索规范（必须严格遵守）

- 每次搜索**只传入一个核心关键词**，不要拼接多个关键词
- 关键词要求：简洁、精准、长约 2-5 个词
- 如果需要搜索多个不同主题，**必须分多次调用** search 工具
- 示例：
  - 正确：`{"query": "GPT-4 发布时间"}` 或 `{"query": "Claude 3.5 Sonnet 参数规模"}`
  - 错误：`{"query": "大语言模型发展历程"}`（模型已知，直接使用）

## 分页策略

如数据/要点超过当前布局容量，**主动拆分为多页**：

- 同一主题多个小点需详细解释 → **每点一页**，大标题不变副标题变
- 单页要点超过 6 条 → 拆分
- 几何体内文字估算后超出边界 → 拆分或精简内容
- 数据对比项过多 → 拆分专题页
